# APK Formats and Installation

## Format Overview

| Format | Extension | Structure | Install Method |
|--------|-----------|-----------|---------------|
| APK | `.apk` | Single monolithic package | `pm install` or PackageInstaller |
| APKM | `.apkm` | ZIP with split APKs | Extract ZIP → install splits via session API |
| XAPK | `.xapk` | ZIP with APKs + manifest.json + OBB | Same as APKM + copy OBB data |
| APKS | `.apks` | ZIP generated by bundletool | Same as APKM |
| AAB | `.aab` | Google's publishing format | Cannot be directly sideloaded |

## Installation Flow in HyperOS Updater

### Single APK (`.apk`)
1. Download to `filesDir/downloads/`
2. Try Shizuku: pipe APK via stdin to `pm install -r -d`
3. Fallback: open system PackageInstaller via Intent

### Split Bundle (`.apkm`, `.xapk`, `.apks`)
1. Download to `filesDir/downloads/`
2. Extract ZIP → individual APK files
3. Try Shizuku: `pm install-create` → `install-write` (per split) → `install-commit`
4. Fallback: install base.apk via PackageInstaller

### APKM Specifics
APKM is APKMirror's proprietary format. It is a ZIP containing:
- `base.apk` — the base application
- `split_config.arm64_v8a.apk` — architecture-specific code
- `split_config.xxxhdpi.apk` — density-specific resources
- `split_config.{language}.apk` — language resources
- ... up to 34+ split APKs

The bundle must be installed as a complete set — installing only `base.apk` fails.
